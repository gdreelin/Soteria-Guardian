# Contributing to Soteria Guardian

## Principles
- Safety-critical behavior must remain deterministic and testable.
- Do not claim medical-device certification or guaranteed fall detection.
- Never commit credentials, tokens, private camera URLs, or resident data.
- Accessibility and privacy are required design constraints.
- New vendor integrations should use adapters rather than tightly coupling the core platform.

## Workflow
1. Create or select an issue.
2. Create a focused branch.
3. Add tests and documentation.
4. Run type checking, linting, and relevant integration tests.
5. Open a pull request describing safety, privacy, and user impact.
